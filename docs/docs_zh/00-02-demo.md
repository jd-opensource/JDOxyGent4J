# 如何运行demo？

使用Maven直接运行我们的任意一个demo示例。

### 为什么下载后直接运行demo系统不启动？
---
OxyGent只是智能体系统框架，本身不提供LLM服务。因此您需要在环境变量中设置自己的LLM API。
```bash
export OXY_LLM_API_KEY="your-api-key"
export OXY_LLM_BASE_URL="your-llm-endpoint"
export OXY_LLM_MODEL_NAME="your-model-name"
```

### 快速开始示例

最简单的方式是运行我们的[快速开始示例](../../oxygent-core/src/main/java/com/jd/oxygent/core/oxygent/samples/examples/DemoInReadme.java)，它展示了OxyGent的核心功能：

这个示例包含了：
- 多模态LLM配置
- 时间工具、文件工具、数学工具
- 多智能体协作（时间代理、文件代理、数学代理）
- 主控智能体协调

### 为什么我填好了环境变量，但是运行demo时报404？
---
首先，请您检查是否引入了不存在的环境变量。
如果不能解决问题，请查看[关于llm api](./01-03-select_llm.md)。

### 系统启动和管理

如果您想了解MAS系统的底层启动和管理机制，可以参考 [DemoLaunchMas.java](../../oxygent-core/src/main/java/com/jd/oxygent/core/oxygent/samples/examples/backend/DemoLaunchMas.java) 示例，它展示了：

- 手动创建和初始化MAS实例
- 直接调用工具和智能体
- 批量处理多个请求
- CLI模式启动
- Web服务启动
